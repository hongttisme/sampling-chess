# Bottisme — Web Play UI

Front-end for the chess bot, generated by Claude Design and dropped here
verbatim. Pure static (HTML + CSS + React via Babel + chess.js v0.10.3).

## Run locally (zero deps)

```bash
python3 -m http.server 8000 --directory web
# open http://localhost:8000/Bottisme.html
```

The bot in `engine.jsx` is currently a piece-square heuristic (mock for
the design preview). Friend can play against it right now over LAN by
swapping `localhost` for the host IP.

## Wire the real bot (when training finishes)

`engine.jsx` exposes one entry point used by the rest of the app:

```js
window.BotEngine.analyze(chess, temperature) -> {
  vCurrent: number,                 // V at current position [-1, 1] from white POV
  scored: [{move, vAfter, prob, score}, ...],
  topMoves: [...same...],
}
window.BotEngine.sampleMove(scored, temperature) -> {move, ...}
```

To swap in the trained ChessTransformer, replace the body of `analyze` with
a `fetch('/api/analyze', {method: 'POST', body: JSON.stringify({fen, temperature})})`
call and stand up a FastAPI backend that:

1. Loads the latest `ckpt_*.pkl` from `checkpoints/`
2. JIT-compiles `apply_legal_mask(model.apply(...))` once at startup
3. On each `/api/analyze` request: parses FEN, encodes via `sampling_chess.board`,
   runs the net, returns `vCurrent + scored` in the schema above

`scored[i].vAfter` from white POV requires running the net on the position
*after* each candidate move (or just one forward + per-move score from the
policy if you don't need vAfter). For "best possible chess bot", do the
per-move forward pass — gives accurate vAfter for the analysis side panels.

The JSX can stay untouched; only the body of `engine.jsx`'s `analyze`
needs to change.
